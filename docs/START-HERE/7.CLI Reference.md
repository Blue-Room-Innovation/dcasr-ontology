## About `config.yml` (important)

This CLI **can work in explicit mode** (passing files via flags) or in **declarative mode**, using `config.yml`.

`config.yml` allows you to:

- Define **SHACL validation scenarios**
- Define **conversion scenarios** (SHACL → JSON Schema, JSON-LD context, JSON → TS)
- Run **all scenarios at once** without repeating flags
- Centralize paths, versions, and conventions

When **no scenario or overrides are specified**, the CLI:

- Executes **all configured scenarios**
- Uses the **default values** from `config.yml`

## Validate Commands

### `validate owl`

Validates OWL ontologies, with optional merging and reasoning.

```bash
node docker/docker.js run cli validate owl [OPTIONS]
```

**Options:**

- `-i, --inputs INPUTS_CSV`
  CSV list of TTL files
- `--include-codelists`
  Includes `codelists/v*/` in auto-discovery
- `--no-auto`
  Disables auto-discovery
- `-r, --reasoner {HermiT,ELK,none}`
  OWL reasoner
- `-p, --profile {DL,OWL2}`
  OWL profile
- `--build-dir BUILD_DIR`
  Build directory (versioned by default)
- `-m, --merged MERGED`
  Merged OWL file
- `-o, --output OUTPUT`
  Reasoned OWL file
- `-q, --quiet`
  Minimal output

---

### `validate shacl`

Validates RDF data against SHACL.
It can work **by scenario** or **with explicit files**.

```bash
node docker/docker.js run cli validate shacl [SCENARIO] [OPTIONS]
```

**Options:**

- `--list`
  Shows all scenarios configured in `config.yml`
- `-d, --data DATA`
  RDF data file (scenario override)
- `-s, --shapes SHAPES`
  SHACL file (scenario override)
- `-e, --extras EXTRAS_CSV`
  Additional TTL files
- `-f, --format {human,text,turtle,json-ld}`
  Report format

**Behavior:**

- No scenario and no flags → validates **all scenarios**
- With scenario → validates only that one
- With `--data` / `--shapes` → one-off execution

```bash

```

## Convert Commands

### `convert json-schema`

Converts SHACL to JSON Schema.
Works **by scenario**, **with overrides**, or **for all scenarios**.

```bash
node docker/docker.js run cli convert json-schema [SCENARIO] [OPTIONS]
```

**Options:**

- `--list`
  List available scenarios
- `-i, --input INPUT`
  Input SHACL
- `-o, --output OUTPUT`
  Output JSON Schema
- `--naming {curie,local,context}`
  Property naming strategy
- `--context CONTEXT_FILE`
  JSON-LD context (required if `naming=context`)
- `-v, --verbose`
